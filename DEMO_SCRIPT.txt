[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture. When a farmer submits symptoms, the data passes through our Security Checkpoint first. This node scrubs personal info like phone numbers and checks for injection attacks. 

Once cleared, the Diagnosis Agent analyzes the symptoms, using Model Context Protocol tools to query local databases and assess weather risk patterns. Then, the Treatment Agent constructs a tailored cure roadmap, prioritizing organic remedies. Before anything is finalized, the workflow pauses at the Farmer Review node, waiting for a human decision to approve or adjust the plan.
